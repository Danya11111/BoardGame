# RUNBOOK: запуск и демонстрация BoardGame

## 1. Предусловия
- Java 17
- Maven 3.6+
- PostgreSQL (локально)
- Файл `lib/lsfusion-server-6.1.jar` (скачать через `download-server.ps1`)

## 2. Подготовка окружения (один раз)
1. Создать БД:
   - `psql -U postgres -f scripts/postgres-init.sql`
2. Скопировать конфиг:
   - `copy lsfusion.properties.example lsfusion.properties`
3. Заполнить параметры подключения к БД в `lsfusion.properties`.
4. (Необязательно) `boardgame.initialAdminEmail=ваш@email` — пользователь с таким email получит роль администратора при регистрации или при смене email.
5. (Необязательно) `boardgame.registrationUrl=https://...` — полный URL страницы регистрации в браузере; тогда кнопка «Зарегистрироваться» на приветствии откроет его (удобно для веб-клиента).
6. Убедиться, что PostgreSQL запущен и принимает подключения на `db.server` из конфигурации (по умолчанию `localhost:5432`).

## 3. Запуск приложения
1. Из корня проекта:
   - `.\run.ps1` (или `./run.sh` на Linux/macOS)
2. Проверить в консоли, что сервер успешно стартовал:
   - `Logics instance has successfully started`
   - `Server has successfully started`

## 4. Сценарий гостя
1. В навигаторе открыть **«Клуб настольных игр»** (`welcomeLanding`): заголовок, короткое описание, пример с Уно, две кнопки внизу.
2. **«Войти»** — стандартный диалог lsFusion.
3. **«Зарегистрироваться»** — если задан `boardgame.registrationUrl`, откроется браузер; иначе — краткое сообщение с подсказкой обратиться к администратору.

## 5. Сценарий участника клуба (вошёл как обычный пользователь)
1. В навигаторе открыть **«Главная»** (`userHomeHub`): два крупных действия с пояснениями под ними.
2. **«Найти похожую игру»** — форма с подсказками сверху; сузить список фильтрами, выбрать игру, затем **«Похожие игры из каталога клуба (топ-3)»** или **«Открыть карточку игры»**.
3. **«Библиотека настольных игр»** — полный список с вводным текстом; фильтры, карточка и рекомендации как выше.
4. Убедиться, что в рекомендациях есть балл и пояснение (в т.ч. «Рекомендовано клубом» при ручной связи в демо-данных).

## 6. Сценарий администратора
1. Войти под учётной записью с ролью `admin` (или зарегистрироваться с email из `boardgame.initialAdminEmail`, если параметр задан).
2. Папка **«Администрирование»**:
   - **«Загрузить демо-данные»** — при необходимости;
   - **«Участники и администраторы»** — назначение/снятие роли администратора;
   - **«Управление играми»**, **«Похожие игры (вручную)»**, справочники жанров, издательств, владельцев экземпляров, механик;
   - **«Справочники (просмотр)»** — только чтение.
3. В карточке игры (из библиотеки или подбора) доступно **«Редактировать игру»**.
4. Главная участника (`userHomeHub`) не содержит админских пунктов; папка администрирования скрыта для не-админов.

## 7. Проверка демо-данных
1. Загрузить демо-данные из админки.
2. В списке игр должны быть **Мафия**, **Шпион**, **Бункер**.
3. Для проверки ручных рекомендаций: в демо задана связь **Мафия → Экивоки** («Рекомендовано клубом» в пояснении).

## 8. Docker
- В `.env`: `BOARDGAME_INITIAL_ADMIN_EMAIL` → `-Dboardgame.initialAdminEmail=...`
- В `.env`: `BOARDGAME_REGISTRATION_URL` → `-Dboardgame.registrationUrl=...` (см. `docker/logics/entrypoint.sh`, `docker-compose.yml`).

## 9. Комментарий по полю изображения
- Для lsFusion 6.1 используется `IMAGEFILE`.

## 10. Проверка после изменений
1. `mvn -q clean compile`
2. Кратко пройти сценарии разделов 4–6.

## 11. Частые проблемы
- Порт `rmi.port` уже занят — остановить второй `java.exe` или сменить порт в `lsfusion.properties`.
- Ошибка подключения к БД — проверить `lsfusion.properties` и запуск PostgreSQL.
- Кнопка «Зарегистрироваться» не открывает браузер — проверьте `boardgame.registrationUrl` и среду (на сервере без GUI `Desktop.browse` может быть недоступен).
