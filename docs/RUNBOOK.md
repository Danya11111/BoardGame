# RUNBOOK: запуск и демонстрация BoardGame

## 1. Предусловия
- Java 17
- Maven 3.6+
- PostgreSQL (локально)
- Файл `lib/lsfusion-server-6.1.jar` (скачать через `download-server.ps1`)

## 2. Подготовка окружения (один раз)
1. Создать БД:
   - `psql -U postgres -f scripts/postgres-init.sql`
2. Скопировать конфиг:
   - `copy lsfusion.properties.example lsfusion.properties`
3. Заполнить параметры подключения к БД в `lsfusion.properties`.
4. Убедиться, что в `lsfusion.properties` есть **`settings.enableUI=2`** и **`settings.enableAPI=1`** (как в `lsfusion.properties.example`). Иначе веб-клиент покажет экран логина раньше приветствия.
5. (Необязательно) `boardgame.initialAdminEmail=ваш@email` — пользователь с таким email получит роль администратора при регистрации, при входе (если в вашей сборке lsFusion поддерживается событие входа в логике проекта) и при смене email.
6. (Необязательно) `boardgame.registrationUrl=https://...` — полный URL страницы регистрации в браузере. Если URL задан, кнопка «Зарегистрироваться» на приветствии откроет его. Если не задан, пользователь увидит спокойное сообщение: обратиться к администратору за логином и нажать «Войти»; администратору подсказано настроить `boardgame.registrationUrl`.
7. Убедиться, что PostgreSQL запущен и принимает подключения на `db.server` из конфигурации (по умолчанию `localhost:5432`).

## 3. Запуск приложения
1. Из корня проекта:
   - `.\run.ps1` (или `./run.sh` на Linux/macOS)
2. Проверить в консоли, что сервер успешно стартовал:
   - `Logics instance has successfully started`
   - `Server has successfully started`

## 4. Сценарий гостя
1. Открыть веб-клиент: при **`settings.enableUI=2`** первым экраном должна быть форма **«Клуб настольных игр»** (`welcomeLanding` в навигаторе): заголовок, короткое описание, пример с Уно, подсказка про вход и главную, кнопки **«Войти»** и **«Зарегистрироваться»**.
2. **«Войти»** — стандартный диалог lsFusion (до этого логин не показывается как обязательный первый шаг). После успешного входа открывается **«Главная»** (`userHomeHub`) с двумя сценариями: «Найти похожую игру» и «Библиотека настольных игр».
3. **«Зарегистрироваться»** — при заданном `boardgame.registrationUrl` открывается системный браузер (десктоп) и показывается сообщение со ссылкой (удобно для веб-клиента); при отсутствии URL — понятное сообщение участнику и подсказка администратору про `boardgame.registrationUrl`.

## 5. Сценарий участника клуба (вошёл как обычный пользователь)
1. После входа по умолчанию доступна **«Главная»** (`userHomeHub`): два крупных действия с пояснениями под ними.
2. **«Найти похожую игру»** — форма с подсказками сверху; сузить список фильтрами, выбрать игру, затем **«Показать похожие игры»** или **«Открыть карточку игры»**.
3. **«Библиотека настольных игр»** — полный список с вводным текстом; фильтры, карточка и рекомендации как выше.
4. Убедиться, что в рекомендациях есть балл и пояснение (в т.ч. «Рекомендовано клубом» при ручной связи в демо-данных).

## 6. Сценарий администратора
1. Войти под учётной записью с ролью `admin`. Первого администратора можно получить, задав `boardgame.initialAdminEmail` и зарегистрировавшись или войдя с этим email (плюс смена email у существующей учётки; при поддержке сборкой — также проверка при входе, см. `BoardGameAuth.lsf`).
2. Папка **«Администрирование»** (структура может быть вложенной):
   - **«Каталог игр»**: **«Игры клуба»**, **«Похожие игры для рекомендаций»**;
   - **«Участники и роли»** — назначение и снятие роли администратора (на экране есть напоминания: нельзя снять с себя и с последнего администратора);
   - **«Справочники»** — жанры, издательства, владельцы, механики;
   - **«Демо и проверка»** — загрузка демо-данных;
   - **«Справочники: только просмотр»** — чтение без редактирования.
3. В карточке игры (из библиотеки или подбора) у администратора доступно **«Редактировать игру»**.
4. Главная участника (`userHomeHub`) не содержит админских пунктов; папка администрирования скрыта для не-админов.

## 7. Проверка демо-данных
1. Загрузить демо-данные из админки.
2. В списке игр должны быть **Мафия**, **Шпион**, **Бункер**.
3. Для проверки ручных рекомендаций: в демо задана связь **Мафия → Экивоки** («Рекомендовано клубом» в пояснении).

## 8. Docker
- Перед `docker compose build` должен лежать **`lib/lsfusion-server-6.1.jar`** (скрипт `download-server.ps1` в корне репозитория). Иначе сборка сервиса **logics** завершится ошибкой: образ намеренно не качает JAR при каждой сборке.
- В `.env`: `BOARDGAME_INITIAL_ADMIN_EMAIL` → `-Dboardgame.initialAdminEmail=...`
- В `.env`: `BOARDGAME_REGISTRATION_URL` → `-Dboardgame.registrationUrl=...` (см. `docker/logics/entrypoint.sh`, `docker-compose.yml`).
- В `.env`: **`LSFUSION_LOGICS_HOST`** / **`LSFUSION_LOGICS_PORT`** — хост и порт RMI для контейнера **web** (по умолчанию `logics` и `7652` внутри compose). Если веб-клиент не в той же docker-сети, что logics, задайте, например, `host.docker.internal` или `127.0.0.1` по ситуации (см. `README.md`).

## 9. Комментарий по полю изображения
- Для lsFusion 6.1 используется `IMAGEFILE`.

## 10. Проверка после изменений
1. `mvn -q clean compile`
2. Кратко пройти сценарии разделов 4–6.

## 11. Частые проблемы
- Порт `rmi.port` уже занят — остановить второй `java.exe` или сменить порт в `lsfusion.properties`.
- Ошибка подключения к БД — проверить `lsfusion.properties` и запуск PostgreSQL.
- Кнопка «Зарегистрироваться» не открывает браузер — проверьте `boardgame.registrationUrl` и среду (на сервере без GUI `Desktop.browse` может быть недоступен); во веб-клиенте после действия показывается текст со ссылкой.
- **`UnknownHostException: logics`** у веб-клиента — Tomcat резолвит имя сервиса compose с хоста, где это имя не существует. Настройте `LSFUSION_LOGICS_HOST` для сервиса `web` или используйте `127.0.0.1` в дескрипторе контекста локального Tomcat (см. `README.md`).
- Сразу логин вместо приветствия — на сервере логики не задано `settings.enableUI=2` (локально: `lsfusion.properties`; Docker: уже в `docker/logics/entrypoint.sh`).
